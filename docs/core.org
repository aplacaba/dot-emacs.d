* Core Module (~config/core.el~)

Core editor settings and the completion stack.

** Built-in Settings

- =y-or-n-p= alias for =yes-or-no-p=
- Trailing whitespace deleted on save
- Lock files, auto-save, and backup files disabled
- =tab-always-indent= set to =complete= (tab completes at point)
- Initial scratch buffer uses =org-mode=
- Auth sources read from =~/.authinfo=
- Custom file set to =~/.emacs.d/custom.el= (gitignored)

** Magit

Full-featured git porcelain with transient level 7 (all commands visible).

| Key            | Command        |
|----------------+----------------|
| =C-M-g=        | magit-status   |
| =C-x g=        | magit-status   |

** Vterm

Terminal emulator using libvterm. Skipped on Windows (=is-windows= check).

- 10,000 line scrollback
- Kill buffer on exit
- =hl-line-mode= disabled inside vterm

| Key            | Command                |
|----------------+------------------------|
| =s-t=          | vterm-toggle           |
| =C-M-t=        | vterm-toggle-cd        |
| =C-c t=        | vterm-copy-mode        |
| =C-q=          | vterm-send-next-key    |

** Completion Stack

Vertico (vertical minibuffer), Corfu (in-buffer popup), Orderless (flexible matching), Consult (search/switch commands), Cape (completion extensions), Tempel (templates).

*** Vertico

Vertical minibuffer completion. Enabled globally.

*** Corfu

In-buffer popup completion.

- =corfu-auto t= — popup triggers automatically
- =corfu-quit-no-match 'separator= — dismiss when no match after separator

*** Orderless

Space-separated component matching with = &[]= as component separators.

*** Consult

| Key     | Command               |
|---------+-----------------------|
| =C-x b= | consult-buffer        |
| =C-c s= | consult-ripgrep       |
| =C-c h= | consult-history       |
| =C-s=   | consult-line-literal  |

=consult-line-literal= uses substring matching (not orderless), making it a practical isearch replacement.

*** Cape

Prefix key =C-c p= provides completion extensions (dabbrev, file, elisp-block).

*** Tempel

Template expansion via =M-+= (complete) and =M-*= (insert). Uses =tempel-collection= for ready-made templates. Hooked into =prog-mode=, =conf-mode=, and =text-mode=.

** Perspective

Workspace/session management via =perspective-mode=.

| Key            | Command            |
|----------------+--------------------|
| =C-x C-b=      | persp-ibuffer      |
| =C-c C-.=      | perspective prefix |

** Other Packages

- =which-key= — shows available keybindings
- =exec-path-from-shell= — inherits shell PATH (skipped on Windows)
- =dabbrev= — dynamic abbreviation with =M-/=
