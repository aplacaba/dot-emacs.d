* Programming Module (~config/programming.el~)

Language support, linting, and developer tools.

** Editor Defaults

- =electric-pair-mode= — auto-close delimiters
- =global-hl-line-mode= — highlight current line
- =indent-bars= — indentation guides for YAML and Python modes

** Languages

*** Web / JS

- =web-mode= for =.tsx=, =.jsx=, =.html=, =.css=, =.scss= (2-space indent)
- =json-mode= with 2-space JS indent
- =js2-mode= for JavaScript
- =emmet-mode= hooked into =web-mode=, =css-mode=, =sgml-mode=

*** Clojure

- =cider= for REPL-driven development
- =clojure-mode= with =cider-mode= hook

*** Ruby

- =ruby-ts-mode= for =.rb=, =Rakefile=, =Gemfile=
- =ruby-end= for auto-inserting =end= keywords
- Tree-sitter navigation: =C-c r b= (beginning of defun), =C-c r e= (end of defun)

*** Elixir

- =elixir-ts-mode= with =ruby-end-mode= adapted for =do/= =end= blocks

*** Nix

- =nix-mode= for =.nix= files

*** YAML

- =yaml-mode= for =.yml= / =.yaml=

** Tree-sitter

Grammars listed in =treesit-language-source-alist=. After first install or updates, evaluate:

#+BEGIN_SRC elisp
(mapc #'treesit-install-language-grammar (mapcar #'car treesit-language-source-alist))
#+END_SRC

=treesit-auto= remaps major modes to their tree-sitter variants automatically.

Available grammars: bash, cmake, css, elisp, go, html, javascript, json, make, markdown, python, toml, tsx, typescript, elixir, dockerfile, ruby, yaml.

** Flycheck

Global syntax checking. Custom =cfn-lint= checker for CloudFormation YAML. Requires =cfn-lint= (~pip install cfn-lint~) installed at =~/.local/bin/cfn-lint=.

** AI (gptel)

Gemini integration via =gptel=. API key resolved from =GEMINI_KEY= env var or =~/.authinfo= (host =gemini=, user =api-key=). Default model: =gemini-3-flash-preview=. Responses use =org-mode=.

** Other Tools

- =yasnippet= — global snippet expansion
- =devdocs= — documentation lookup (=C-h d=)
- =markdown-mode= — GitHub-Flavored Markdown for READMEs
