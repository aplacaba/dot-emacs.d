* Emacs Config

A modular Emacs configuration targeting Fedora Linux (primary), macOS 15, and Windows 11.

** Features

- Completion stack: Vertico + Corfu + Orderless + Consult + Cape + Tempel
- Git: Magit with transient level 7
- Terminal: ghostel (libghostty-vt)
- Navigation: Avy, switch-window, ripgrep, consult-line
- Programming: Tree-sitter, flycheck, yasnippet, LSP-ready
- Languages: TypeScript/JSX, Clojure (CIDER), Ruby, Elixir, Python, Common Lisp (SLIME), Nix, YAML, Terraform, Docker, CloudFormation
- Notes: org-mode + denote
- AI: gptel (Gemini)
- Theme: Modus themes with doom-modeline

** Requirements

*** Linux (Fedora)

#+BEGIN_SRC sh
sudo dnf install ripgrep
#+END_SRC

*** macOS

#+BEGIN_SRC sh
brew install ripgrep roswell
#+END_SRC

*** Windows

- ghostel is skipped on Windows automatically (it supports macOS/Linux/FreeBSD only); use eshell there. The native module downloads automatically on first use — no system dependency required

*** Optional

- =sbcl= — Common Lisp REPL via SLIME
- =cfn-lint= (~pip install cfn-lint~) — CloudFormation linting
- =multimarkdown= — Markdown preview
- Gemini API key in =~/.authinfo= or =GEMINI_KEY= env var for gptel

** Installation

#+BEGIN_SRC sh
git clone <repo-url> ~/.emacs.d
#+END_SRC

First launch will install all packages automatically via =use-package= =:ensure=.

After first launch, install tree-sitter grammars by evaluating:

#+BEGIN_SRC elisp
(mapc #'treesit-install-language-grammar (mapcar #'car treesit-language-source-alist))
#+END_SRC

** Architecture

This is a [[https://en.wikipedia.org/wiki/Literate_programming][literate configuration]] — each module is an org file containing
prose documentation alongside elisp code blocks. =init.el= and =early-init.el=
are thin bootstraps that use =org-babel-load-file= to tangle and load the org
sources.

#+BEGIN_SRC
~/.emacs.d/
├── early-init.el          # bootstrap: loads early-init.org
├── init.el                # bootstrap: package setup, loads config/*.org
├── config/
│   ├── core.org           # built-in settings, completion stack, magit, ghostel
│   ├── rice.org           # themes, fonts, modeline
│   ├── nav.org            # avy, switch-window, ripgrep
│   ├── notes.org          # org-mode, denote
│   ├── devops.org         # terraform, docker, cfn, pg/pgmacs
│   ├── programming.org    # languages, tree-sitter, flycheck, yasnippet, gptel
│   ├── llisp.org          # SLIME/SBCL
│   ├── bindings.org       # global keybindings (Colemak-aware)
│   ├── mac.org            # macOS overrides
│   └── windows.org        # Windows overrides
├── snippets/              # yasnippet snippets
└── custom.el              # customize settings (gitignored)
#+END_SRC

Each module is a self-documenting org file. To edit the config, edit the
=.org= files — the generated =.el= files are gitignored.

** Customizations

OS-specific config is in =config/mac.org= and =config/windows.org=. Emacs =customize= settings are stored in =custom.el= (gitignored).
